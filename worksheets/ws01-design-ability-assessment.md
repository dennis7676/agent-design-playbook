# Worksheet 1: Design Ability Self-Assessment

> Companion to [Chapter 1 — Why Design Ability?](../chapters/ch01-why-design-ability.md)

## Before You Start

This worksheet helps you identify where you are on the prompting → coding → design spectrum, and what to focus on next.

---

## Exercise 1: Where Are You Now? (5 min)

Check the statements that describe your current approach:

**Phase 1 — Prompting**
- [ ] I write a new prompt for each task
- [ ] My prompts aren't saved or reusable
- [ ] When output is wrong, I rephrase the prompt
- [ ] I don't have a system — I have conversations

**Phase 2 — Coding/Automating**
- [ ] I have scripts or workflows that run recurring tasks
- [ ] I've built 5+ automations
- [ ] When something breaks, I'm not sure which automation caused it
- [ ] Adding a new automation sometimes breaks existing ones

**Phase 3 — Designing**
- [ ] I have a document that governs how all agents behave
- [ ] I document *why* I chose one approach over another
- [ ] I can explain how my components connect to each other
- [ ] When I add something new, I check what it affects

**Your dominant phase**: The section with the most checks is where you currently operate.

## Discussion Prompts (Study Group)

1. **"When did you shift from Phase 1 to Phase 2?"** What triggered the transition? Was it a failure, a scaling need, or curiosity?

2. **"What's the most expensive mistake your AI system has made?"** How would a design approach have prevented it?

3. **"If you had to explain your AI system to a new team member in 5 minutes, could you?"** What would you struggle to explain? Those gaps are design gaps.

---

## Key Takeaway

The Phase 1→2→3 transition isn't about tools or techniques. It's about shifting from *"How do I get this task done?"* to *"How do I design a system that handles this class of tasks?"*

Your next action should target whatever phase you're currently in — not skip ahead.
