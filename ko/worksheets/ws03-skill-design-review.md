# 워크시트 3: 스킬 설계 리뷰

> [3장 -- 스킬 아키텍처](../chapters/ch03-skill-architecture.md) 함께 보기

## 시작하기 전에

이 워크시트는 3장의 5가지 원칙을 활용하여 기존 스킬을 평가하거나 새로운 스킬을 설계하는 데 도움을 줍니다.

---

## 실습 1: 필요성 테스트 (스킬당 5분)

스킬 3개를 골라 보세요 (자기 것이든 템플릿이든). 각 스킬에 대해:

| 스킬 이름 | 없애면 뭐가 깨지나? | 판정 |
|----------|---------------------|------|
| | | 유지 / 통합 / 보관 |
| | | 유지 / 통합 / 보관 |
| | | 유지 / 통합 / 보관 |

**위험 신호**: "깨지는 게 없다" 또는 "다른 스킬이 처리한다" -> 보관(Archive) 후보입니다.

---

## 실습 2: 트리거 정밀도 점검 (10분)

스킬의 현재 트리거를 적고, 평가해 보세요:

**현재 트리거**: `______________________________`

| 점검 항목 | 통과? |
|----------|-------|
| 단일 조건인가 (OR로 연결되지 않은)? | [ ] |
| "이럴 때는 아님" 섹션이 있는가? | [ ] |
| 인접 스킬과 구분이 되는가? | [ ] |
| 처음 보는 에이전트도 정확히 해석할 수 있는가? | [ ] |

**하나라도 통과 못 했다면** 트리거를 다시 작성하세요:

**개선된 트리거**: `______________________________`

**이럴 때는 아님**: `______________________________`

---

## 실습 3: 주의사항(Gotcha) 점검 (15분)

이 스킬이 잘못될 수 있는 모든 경우를 나열하세요:

| 주의사항 | 왜 발견하기 어려운지 | 해결 방법 | 해당 안 되는 경우 |
|---------|--------------------|-----------|--------------------|
| 1. | | | |
| 2. | | | |
| 3. | | | |

**기준선**: 잘 만든 스킬은 문서화된 주의사항이 3개 이상입니다. 그보다 적다면 아직 스킬을 충분히 운영해 보지 않았거나 실패를 추적하지 않은 것입니다.

---

## 실습 4: 직무기술서 테스트 (10분)

스킬에 대해 다섯 가지 질문에 답해 보세요:

1. **역할** -- 이 스킬이 하는 일을 한 문장으로?
   > _______________________________________

2. **책임 범위** -- 다른 스킬과 겹치는 부분이 있는가?
   > _______________________________________

3. **성공 기준** -- 제대로 동작했는지 어떻게 아는가?
   > _______________________________________

4. **에스컬레이션** -- 범위 밖의 상황을 만나면 어떻게 하는가?
   > _______________________________________

5. **온보딩** -- 처음 보는 에이전트가 설명만 읽고 사용할 수 있는가?
   > _______________________________________

**채점**: 5/5 명확 = 잘 설계됨. 3-4개 = 다듬기 필요. 3개 미만 = 재설계 필요.

---

## 실습 5: 새 스킬 만들기 (20분)

[SKILL.md 시작 템플릿](../templates/SKILL-md-starter.md)을 활용하여:

1. 트리거 정의 (단일 조건으로)
2. "이럴 때는 아님" 섹션을 먼저 작성
3. 정상 동작 시나리오를 쓰기 전에 예상 주의사항 3개를 먼저 나열
4. 검증 가능한 출력 정의 (기계적으로 확인할 수 있는 것은?)
5. 리뷰 일정 설정 (이 스킬을 언제 점검할 것인가?)

---

## 토론 주제 (스터디 그룹)

1. **"가장 안 좋은 스킬을 보여주세요."** 뭐가 문제인가요? 그룹에서 실패 패턴을 짚어낼 수 있나요?

2. **"주의사항 대 예시 -- 비율이 어떻게 되나요?"** 그룹 내에서 비교해 보세요. 누가 어디에 토큰을 더 쓰고 있나요?

3. **"스킬을 보관 처리한 적 있나요?"** 보관의 계기가 뭐였나요? 어떤 신호를 보고 결정했나요?
