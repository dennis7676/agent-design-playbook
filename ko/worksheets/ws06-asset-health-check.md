# 워크시트 6: 자산 건강 점검

> [6장 — 자산 관리](../chapters/ch06-asset-management.md) 함께 보기

## 시작하기 전에

이 워크시트는 현재 보유한 자산의 건강 상태를 실용적으로 점검합니다. 매월(또는 "뭔가 어수선하다" 싶을 때) 돌려서, 드리프트가 부채로 커지기 전에 잡으세요.

---

## 실습 1: 자산 인벤토리 (15분)

보유한 AI 자산을 전부 나열하세요. 빠짐없이 — 잊고 있던 것이 보통 문제의 원인입니다.

| # | 자산 이름 | 유형 | 마지막 사용일 | 마지막 수정일 | 상태 |
|---|---------|------|------------|------------|------|
| 1 | | skill / workflow / command / template / integration | | | active / stale / unknown |
| 2 | | | | | |
| 3 | | | | | |
| 4 | | | | | |
| 5 | | | | | |
| ... | | | | | |

**집계**: 전체 ___개. Active ___개. Stale ___개. Unknown ___개.

---

## 실습 2: 노후화 점검 (10분)

"stale" 또는 "unknown"으로 표시된 각 자산에 대해:

| 자산 | 마지막 사용일 | 아직 필요한가? | 조치 |
|------|------------|-------------|------|
| | | 예 → 재활성화 / 아니오 → 아카이브 / 모르겠음 → 테스트 | |
| | | | |
| | | | |

**30일 규칙**: 30일 이상 미사용이고 계절성 작업이 아니라면, 아카이브 후보입니다.

---

## 실습 3: 의존성 맵 (15분)

가장 중요한 자산 5개의 의존 관계를 그려 보세요:

```
[자산 A] ──의존──→ [자산 B]
[자산 A] ──에 의존됨──→ [자산 C]
```

| 자산 | 의존 대상 | 의존하는 쪽 | 변경 시 위험도 |
|------|---------|-----------|-------------|
| | | | 낮음 / 보통 / 높음 |
| | | | |
| | | | |
| | | | |
| | | | |

**고위험 자산** (의존하는 쪽이 3개 이상): 수정 전 반드시 백업 프로토콜을 적용하세요.

---

## 실습 4: 중복 탐지 (10분)

비슷한 역할을 하는 것 같은 자산을 나열하세요:

| 자산 A | 자산 B | 중복도 | 해결 방안 |
|--------|--------|-------|----------|
| | | __% | 통합 / 역할 분리 / 하나 아카이브 |
| | | | |
| | | | |

**검증**: 두 자산을 다른 사람에게 설명했을 때, "둘이 뭐가 다른 거예요?"라는 질문이 나온다면 중복입니다.

---

## 실습 5: 일관성 점검 (10분)

아래 질문에 솔직하게 답하세요:

| 점검 항목 | 상태 |
|----------|------|
| 모든 자산이 동일한 네이밍 규칙을 따르나요? | [ ] 예 [ ] 부분적 [ ] 아니오 |
| 모든 자산이 현재 스키마/형식을 사용하나요? | [ ] 예 [ ] 부분적 [ ] 아니오 |
| 아무 자산이나 골라서 다른 자산과의 관계를 설명할 수 있나요? | [ ] 예 [ ] 대체로 [ ] 아니오 |
| 서로 모순되는 의사결정 기록이 있나요? | [ ] 없음 [ ] 일부 [ ] 많음 |
| 새로운 사람이 문서만 보고 온보딩할 수 있나요? | [ ] 예 [ ] 부분적 [ ] 아니오 |

**채점**: "예" 5개 = 건강. 3~4개 = 자연스러운 드리프트. 3개 미만 = 일관성 감사 필요.

---

## 실습 6: 건강 점수 계산 (5분)

시스템 건강 점수를 계산하세요:

| 지표 | 값 | 가중치 | 점수 |
|------|---|--------|------|
| 활성 비율 (active / 전체) | __% | x 0.30 | |
| 최신성 (30일 내 수정 / 전체) | __% | x 0.25 | |
| 비중복 비율 (중복 없는 자산 / 전체) | __% | x 0.20 | |
| 의존성 문서화 (매핑 완료 / 전체) | __% | x 0.15 | |
| 네이밍 일관성 (규칙 준수 / 전체) | __% | x 0.10 | |
| **총 건강 점수** | | | **__/100** |

**해석**:
- 80점 이상 = 건강한 시스템, 현재 관리 주기 유지
- 60~79점 = 자연스러운 드리프트, 정리 일정 잡기
- 60점 미만 = 상당한 부채 누적, 개선 작업 우선 필요

## 토론 주제 (스터디 그룹)

1. **"건강 점수가 얼마인가요?"** 그룹 내에서 비교해 보세요. 낮은 점수의 가장 큰 원인은 무엇인가요?

2. **"대규모 정리를 해 본 사람?"** 무엇을 아카이브했나요? 기분이 어땠나요? 뭔가 깨졌나요?

3. **"자산이 몇 개쯤 됐을 때 처음 벅차다고 느꼈나요?"** 그룹 멤버들의 스케일링 한계를 비교해 보세요. 비슷한가요?
