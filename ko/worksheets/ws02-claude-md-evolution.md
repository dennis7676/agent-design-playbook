# 워크시트 2: CLAUDE.md 발전 실습

> [2장 -- CLAUDE.md 설계](../chapters/ch02-claude-md-design.md) 함께 보기

## 시작하기 전에

이 워크시트는 3계층 모델(지도 -> 가드레일 -> 인지)을 활용하여 CLAUDE.md를 새로 만들거나 기존 것을 점검하는 과정을 안내합니다.

---

## 실습 1: 1계층 -- 지도 (15분)

프로젝트에 맞게 빈칸을 채워 보세요:

### 워크스페이스 구조
```markdown
## 폴더 구조
| 폴더 | 용도 |
|------|------|
| [폴더1] | [어떤 내용이 들어가는지] |
| [폴더2] | [어떤 내용이 들어가는지] |
| [폴더3] | [어떤 내용이 들어가는지] |
```

### 도구 역할
```markdown
## 도구 역할
| 도구 | 역할 |
|------|------|
| [도구1] | [주요 책임] |
| [도구2] | [주요 책임] |
```

### 주요 파일 위치
```markdown
## 주요 파일
| 파일 | 경로 | 용도 |
|------|------|------|
| [설정] | [경로] | [무엇을 제어하는지] |
| [데이터] | [경로] | [무엇을 저장하는지] |
```

**체크포인트**: 처음 보는 에이전트가 이것만 읽고 모든 것의 위치를 파악할 수 있나요? 그렇지 않다면 무엇이 빠져 있나요?

## 실습 2: 2계층 -- 가드레일 (15분)

### 리스크 분류

에이전트가 수행하는 행동을 떠올려 보세요. 각 행동을 위험 등급으로 분류하세요:

| 행동 | 5분 내 되돌릴 수 있나? | 워크스페이스 바깥에 영향? | 위험 등급 |
|------|----------------------|-------------------------|----------|
| [행동1] | 예 / 아니오 | 예 / 아니오 | L__ |
| [행동2] | 예 / 아니오 | 예 / 아니오 | L__ |
| [행동3] | 예 / 아니오 | 예 / 아니오 | L__ |
| [행동4] | 예 / 아니오 | 예 / 아니오 | L__ |

**판단 기준**:
- 둘 다 "예" + "아니오" -> L1-L2 (자율 실행)
- 하나라도 해당 -> L3-L4 (사전 확인)
- 둘 다 "아니오" + "예" -> L5 (사람 개입 필수)

### 실패 기록

에이전트가 저질렀거나 저지를 수 있는 실수 3가지를 나열하세요:

| 실수 | 무슨 일이 있었나 | 어떤 규칙이 막을 수 있었나 |
|------|-----------------|--------------------------|
| 1. | | |
| 2. | | |
| 3. | | |

**실패 하나가 가드레일 하나가 됩니다.** CLAUDE.md에 넣을 규칙으로 작성해 보세요.

## 실습 3: 3계층 -- 인지 (20분)

### 모호한 입력 시나리오

에이전트가 받을 수 있는 모호한 입력 3가지를 나열하세요:

| 입력 | 가능한 해석 | 에이전트는 어떻게 판단해야 하나? |
|------|-----------|-------------------------------|
| "[모호한 요청 1]" | A: __ / B: __ | |
| "[모호한 요청 2]" | A: __ / B: __ | |
| "[모호한 요청 3]" | A: __ / B: __ | |

각 항목에 대해: 에이전트가 추측해야 하나요, 물어봐야 하나요, 아니면 패턴을 참조해야 하나요? 확신도 임계값은 어느 정도가 적절한가요?

### 의사결정 패턴

에이전트가 반복적으로 내리는 결정 3가지를 떠올려 보세요:

| 결정 유형 | 평소 선택 | 이유 |
|----------|----------|------|
| [예: "도구 선택"] | [예: "가벼운 것부터 시도"] | [예: "과잉 설계 방지"] |
| | | |
| | | |

이런 패턴을 CLAUDE.md에 문서화된 규칙으로 만들 수 있습니다.

---

## 토론 주제 (스터디 그룹)

1. **"시스템에서 가장 중요한 가드레일은 뭔가요?"** 없애면 가장 큰 사고가 날 그 규칙 말입니다.

2. **"지금 모호한 입력을 어떻게 처리하고 있나요?"** 에이전트가 물어보나요, 추측하나요, 멈추나요? 어떤 반응이 가장 유용한가요?

3. **실패 기록을 공유해 보세요.** 그룹 전체에서 어떤 패턴이 보이나요? 모두에게 적용 가능한 범용 가드레일이 있나요?
