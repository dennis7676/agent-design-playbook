# 에이전트 설계 플레이북

> 이론이 아닌, 166개 프로덕션 자산을 2개월간 운영한 실전 경험에서 나온 AI 에이전트 시스템 설계 가이드.

## 왜 만들었나

시장이 양극화되어 있습니다:
- **이론 쪽**: Anthropic의 "Building Effective Agents" eBook, Packt 교과서
- **코드 쪽**: LangGraph 튜토리얼, CrewAI 퀵스타트

빠져 있는 것: **"왜 이 구조를 선택하고 저 구조를 버렸는가?"** — 실제 시스템 뒤의 설계 결정.

이 플레이북은 그 공백을 채웁니다:
- **166개 프로덕션 자산** (스킬 20, 워크플로우 77, 커맨드 25, MCP 서버 7)
- **150+ 아키텍처 의사결정** ADR로 문서화
- **3개 Permanent Note**가 형성하는 핵심 thesis

## 세 가지 핵심 주장

1. **"AI 시대의 실력은 에이전트를 잘 설계하는 것이다"** — 프롬프팅이 아니라, 코딩이 아니라. 에이전트를 효과적으로 만드는 시스템을 설계하는 것.

2. **"시스템적 조직화가 개인기를 이긴다"** — 구조화된 관리 접근법이 즉흥적 천재성을 압도한다.

3. **"큐레이션은 품질 관리가 아닌 서사 설계다"** — 가치는 개별 요소가 좋은 것이 아니라, 전체 시스템이 일관된 이야기를 말하는 것에 있다.
